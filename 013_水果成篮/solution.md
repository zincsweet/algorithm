你正在探访一家农场，农场从左到右种植了一排果树。这些树用一个整数数组 `fruits` 表示，其中 `fruits[i]` 是第 `i` 棵树上的水果 **种类** 。

你想要尽可能多地收集水果。然而，农场的主人设定了一些严格的规矩，你必须按照要求采摘水果：

- 你只有 **两个** 篮子，并且每个篮子只能装 **单一类型** 的水果。每个篮子能够装的水果总量没有限制。
- 你可以选择任意一棵树开始采摘，你必须从 **每棵** 树（包括开始采摘的树）上 **恰好摘一个水果** 。采摘的水果应当符合篮子中的水果类型。每采摘一次，你将会向右移动到下一棵树，并继续采摘。
- 一旦你走到某棵树前，但水果不符合篮子的水果类型，那么就必须停止采摘。

给你一个整数数组 `fruits` ，返回你可以收集的水果的 **最大** 数目。

 

**示例 1：**

```
输入：fruits = [1,2,1]
输出：3
解释：可以采摘全部 3 棵树。
```

**示例 2：**

```
输入：fruits = [0,1,2,2]
输出：3
解释：可以采摘 [1,2,2] 这三棵树。
如果从第一棵树开始采摘，则只能采摘 [0,1] 这两棵树。
```

**示例 3：**

```
输入：fruits = [1,2,3,2,2]
输出：4
解释：可以采摘 [2,3,2,2] 这四棵树。
如果从第一棵树开始采摘，则只能采摘 [1,2] 这两棵树。
```

**示例 4：**

```
输入：fruits = [3,3,3,1,2,1,1,2,3,3,4]
输出：5
解释：可以采摘 [1,2,1,1,2] 这五棵树。
```

 

**提示：**

- `1 <= fruits.length <= 105`
- `0 <= fruits[i] < fruits.length`

```c++
class Solution {
public:
    int totalFruit(vector<int>& fruits) {
        int ret = 0, left = 0, right = 0;
        int sz = fruits.size();
        std::unordered_map<int, int> m;
        for (right; right < sz; ++right) {
            ++m[fruits[right]];
            while (m.size() > 2) {
                if (--m[fruits[left]] == 0) {
                    m.erase(fruits[left]);
                }
                ++left;
            }
            ret = max(right - left + 1, ret);
        }
        return ret;
    }
};
```

解法采用**滑动窗口 + 哈希表**，在 O(n) 时间内找到最多包含两种水果类型的最长连续子数组长度。

### 核心原理

- 问题等价于：寻找最长的连续子数组，其中**不同整数的个数 ≤ 2**（因为只有两个篮子）。
- 维护窗口 `[left, right]`，用哈希表 `m` 记录窗口内每种水果的出现次数。
- 右指针不断右移，将新水果加入窗口。
- 当哈希表大小 > 2 时，说明窗口内出现了第三种水果，需要收缩左边界：移动左指针，并减少对应水果的计数，若计数归零则从哈希表中删除该类型，直到哈希表大小 ≤ 2。
- 每次调整后窗口均合法（最多两种水果），更新最大长度 `ret = max(ret, right - left + 1)`。

### 算法步骤

1. 初始化 `left = 0`，哈希表 `m`，`ret = 0`。
2. 遍历 `right` 从 0 到 `n-1`：
	- `m[fruits[right]]++` 加入当前水果。
	- 若 `m.size() > 2`，则循环移动 `left`：
		- `m[fruits[left]]--`。
		- 若该水果计数变为 0，则 `m.erase(fruits[left])`。
		- `left++`。
	- 此时窗口内水果种类 ≤ 2，更新 `ret = max(ret, right - left + 1)`。
3. 返回 `ret`。

### 关键细节

- **为什么用 `while` 而不是 `if`**：一次左移可能不足以将种类降到 ≤2，必须循环直到条件满足。例如窗口内已有两种水果，加入第三种，需要连续移出直到第三种被完全移出或某种原有水果消失。
- **哈希表的作用**：快速统计窗口内每种水果的数量以及当前种类数。由于水果种类编号范围大（0 到 n-1），但窗口内最多只存 3 种键，空间 O(1)。
- **erase 的必要性**：当某种水果计数归零时，若不删除键，`size()` 仍会计数该种类，导致误判。必须显式删除。
- **更新时机**：先收缩窗口保证合法，再计算长度，确保长度基于合法窗口。
- **复杂度**：每个元素最多入窗口一次、出窗口一次，哈希表操作平均 O(1)，总时间 O(n)，空间 O(1)（最多 3 个键值对）。

### 正确性保证

- 滑动窗口维护了以每个 `right` 为结尾的最长合法子数组（种类 ≤ 2）。遍历所有右端点后，全局最长子数组必然被记录。
