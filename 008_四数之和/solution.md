[18. 四数之和](https://leetcode.cn/problems/4sum/)

中等

给你一个由 `n` 个整数组成的数组 `nums` ，和一个目标值 `target` 。请你找出并返回满足下述全部条件且**不重复**的四元组 `[nums[a], nums[b], nums[c], nums[d]]` （若两个四元组元素一一对应，则认为两个四元组重复）：

- `0 <= a, b, c, d < n`
- `a`、`b`、`c` 和 `d` **互不相同**
- `nums[a] + nums[b] + nums[c] + nums[d] == target`

你可以按 **任意顺序** 返回答案 。

 

**示例 1：**

```
输入：nums = [1,0,-1,0,-2,2], target = 0
输出：[[-2,-1,1,2],[-2,0,0,2],[-1,0,0,1]]
```

**示例 2：**

```
输入：nums = [2,2,2,2,2], target = 8
输出：[[2,2,2,2]]
```

 

**提示：**

- `1 <= nums.length <= 200`
- `-109 <= nums[i] <= 109`
- `-109 <= target <= 109`

## 解题思路（排序 + 双指针，三数之和的推广）

### 核心原理

要在无序数组中找到所有不重复的四元组 `(a,b,c,d)` 使 `a + b + c + d = target`。
先排序，然后**固定前两个数**（`nums[i]` 和 `nums[j]`），将问题转化为在剩余有序区间 `[j+1, n-1]` 中寻找两数之和等于 `target - nums[i] - nums[j]`，即经典的“两数之和”问题，使用双指针 O(n) 求解。

### 算法步骤

1. **排序**
	升序排序 `nums`，这是双指针收缩和去重的基础。
2. **枚举第一个数 `i`**
	- 范围：`0` 到 `n-4`（保证至少剩余三个数）。
	- **去重**：若 `i > 0` 且 `nums[i] == nums[i-1]`，跳过该 `i`，避免重复四元组。
	- 计算剩余三数和的目标：`sub_target = target - nums[i]`（**注意使用 `long long` 防止溢出**，因为 `target` 和 `nums[i]` 均可达 ±1e9）。
3. **枚举第二个数 `j`**
	- 从 `i+1` 到 `n-3`。
	- **去重**：若 `j > i+1` 且 `nums[j] == nums[j-1]`，跳过该 `j`。
	- 计算剩余两数和的目标：`two_target = sub_target - nums[j]`。
4. **双指针查找后两个数**
	- 左指针 `left = j+1`，右指针 `right = n-1`。
	- 当前和 `sum = nums[left] + nums[right]`。
		- 若 `sum < two_target` → 需要增大和 → `left++`。
		- 若 `sum > two_target` → 需要减小和 → `right--`。
		- 若相等 → 找到一组解，记录 `{nums[i], nums[j], nums[left], nums[right]}`，然后：
			- 同时移动 `left++`, `right--`。
			- **跳过重复**：移动后若 `nums[left] == nums[left-1]`，继续 `left++`；若 `nums[right] == nums[right+1]`，继续 `right--`。
5. **循环控制**
	- 完成当前 `j` 的双指针后，`j++` 并跳过后面的重复值。
	- 完成当前 `i` 的所有 `j` 后，`i++` 并跳过后面的重复值。

### 细节与重点

- **溢出处理**：`target - nums[i] - nums[j]` 可能超出 32 位 int 范围，使用 `long long` 进行中间计算。
- **剪枝优化（可选但未在代码中实现）**：
	- 若当前最小四数和 `nums[i] + nums[i+1] + nums[i+2] + nums[i+3] > target`，可提前 `break`。
	- 若当前最大四数和 `nums[i] + nums[n-3] + nums[n-2] + nums[n-1] < target`，可跳过该 `i`（因为 `nums[i]` 太小，后面更大也达不到）。
		用户代码未显式加入这些剪枝，但可自行添加进一步提高效率。
- **去重策略**：
	- 对 `i` 和 `j`：只处理每个值的第一次出现。
	- 对 `left/right`：找到一组解后，跳过与当前 `nums[left]` / `nums[right]` 相等的后续元素，避免重复四元组。
- **边界条件**：
	- `i < sz-3`（至少 4 个数）
	- `j < sz-2`（至少 3 个数）
	- `left < right`（至少 2 个数）

### 复杂度分析

- **时间**：O(n³)
	- 排序 O(n log n)
	- 外层循环 O(n)，内层循环 O(n)，双指针 O(n) → 总 O(n³)
- **空间**：O(1)（不计输出数组），排序使用 O(log n) 栈空间。
