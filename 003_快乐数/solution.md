[202. 快乐数](https://leetcode.cn/problems/happy-number/)

简单

编写一个算法来判断一个数 `n` 是不是快乐数。

**「快乐数」** 定义为：

- 对于一个正整数，每一次将该数替换为它每个位置上的数字的平方和。
- 然后重复这个过程直到这个数变为 1，也可能是 **无限循环** 但始终变不到 1。
- 如果这个过程 **结果为** 1，那么这个数就是快乐数。

如果 `n` 是 *快乐数* 就返回 `true` ；不是，则返回 `false` 。

**示例 1：**

```
输入：n = 19
输出：true
解释：
12 + 92 = 82
82 + 22 = 68
62 + 82 = 100
12 + 02 + 02 = 1
```

**示例 2：**

```
输入：n = 2
输出：false
```

**提示：**

- `1 <= n <= 231 - 1`

```c++
class Solution {
public:
    int op(int n) {
        int result = 0;
        while (n) {
            int num = n % 10;
            result += num*num;
            n /= 10;
        }
        return result;
    }

    bool isHappy(int n) {
        int slow = n, fast = op(n);
        while (slow != fast) {
            slow = op(slow);
            fast = op(op(fast));
        }
        return slow == 1;
    }
};
```

## 解题总结（快慢指针 + 鸽巢原理）

### 平方和的值域有限

对于一个 `d` 位的正整数，最大平方和是：
9^2^+9^2^+⋯+9^2^=81d 

- 例如，一个 10 位数，最大平方和是 810。
- 实际上，对于任意大于 3 位的数，平方和都会迅速变小（比如 9999 → 324，位数减少）。
- 经过研究：对于 32 位整数，平方和不会超过 9^2^×10=810，而且大多数数会掉到 `[1, 243]` 左右。

### 核心思路

- 每个正整数不断进行“各位平方和”运算，最终必然进入循环（鸽巢原理：平方和结果有上界）。
- 用快慢指针检测循环：
	- `slow` 每次走 1 步（一次 `op`）
	- `fast` 每次走 2 步（两次 `op`）
- 若循环为 `1 → 1 → …`，则 `slow` 与 `fast` 会在 1 相遇 → 快乐数。
- 若循环不含 1，则相遇点不为 1 → 非快乐数。

### 算法步骤

1. 定义 `op(n)`：计算 `n` 的各位平方和。
2. 初始化 `slow = n`，`fast = op(n)`。
3. `while (slow != fast)`：
	- `slow = op(slow)`
	- `fast = op(op(fast))`
4. 循环结束时 `slow == fast`，返回 `slow == 1`。

### 关键细节

- **起点差异**：`fast` 初始为 `op(n)` 而非 `n`，确保循环能正常启动（避免初始相等误判）。
- **空间 O(1)**：只用了两个整型变量，无需哈希集合。
- **正确性保证**：鸽巢原理保证必有循环；快慢指针必定在环内相遇；相遇点是否为 1 直接决定答案。
